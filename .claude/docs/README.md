# Docs

Reference documentation: decisions, analysis notes, and architecture context.

## When to create a file here

- Architectural decision records (ADRs)
- Analysis notes produced during investigation (e.g., root-cause exploration)
- Context documents that span multiple tasks (e.g., auth system overview)
- Any documentation that helps future planners or executors understand the system

## Rules

- Do not duplicate what is already in the code or git history
- Keep files scoped — one topic per file
- Update or remove files when they go stale
