# Important Files

Quick reference for navigating, modifying, or extending this codebase.
Each entry notes what the file does, what depends on it, and what to watch out for.

---

## Authentication

### `auth.ts` (root)
NextAuth v5 configuration.
- Configures Google OAuth provider
- `signIn` callback: calls `isEmailAllowed()` — rejects non-whitelisted emails
- `session` callback: attaches `user.role` to every session token
- **Change with care**: modifying callbacks affects all protected routes and session shape

### `lib/auth.ts`
Email whitelist and user query helpers.
- `isEmailAllowed(email)`: queries `users` table — called on every sign-in
- `getUserByEmail(email)`: returns user with role — called in session callback
- Input sanitized (trim, lowercase, regex) before DB query

### `app/providers.tsx`
Wraps the app in `SessionProvider`. Required for `useSession()` in Client Components.

## Database

### `db/schema.ts`
Single source of truth for the DB schema (Drizzle ORM).
- `users`: email whitelist + role enum (admin / presenter / viewer)
- `businessFunctionDemos`: demo content per category (6-value enum)
- `aiAgents`: agent definitions (not yet queried — mocks used instead)
- **Modify here first** when adding/changing tables; then run `npm run db:generate` + `npm run db:push`

### `lib/db.ts`
Drizzle + Neon serverless connection. Import `db` from here for all queries.

### `drizzle.config.ts`
Migration config — schema path, output directory, DB credentials from `DATABASE_URL`.

### `drizzle/`
SQL migration files. Auto-generated by `npm run db:generate`. Do not edit manually.

## Panorama 360° System

### `lib/panorama-hotspots.ts`
All hotspot data and coordinate math.
- `PanoramaHotspot` type: id, title, description, x, y (0–1 normalized), pitch, yaw, mediaUrl, link
- `xToYaw(x)` and `yToPitch(y)`: convert normalized coordinates to Pannellum degrees
- `getHotspotsForScene(slug)`: returns hotspots for a given experience
- **Add/edit hotspots here.** Construction hotspots are mostly commented out.

### `lib/panorama-experiences.ts`
Static metadata for the four 360° experiences (hr, classroom, construction, ecommerce).
- Provides slug, title, description, card image URL, and viewer image path
- **Add new scenes here** before adding hotspots or routes

### `components/panorama/Panorama360View.tsx`
Pannellum viewer wrapper (Client Component, SSR disabled).
- Mounts the equirectangular image via `react-pannellum-next`
- Renders hotspot markers as custom DOM nodes overlaid on the viewer
- `handleHotspotClick(id)`: updates Zustand store → triggers panel + camera
- `lookAt(pitch, yaw, duration)`: animates camera; queued via `pendingLookAt` ref if viewer not yet ready
- **Most fragile file.** Camera timing, hotspot click propagation, and Pannellum API quirks all live here.

### `components/panorama/SceneViewport.tsx`
Full-screen layout orchestrator (Client Component).
- Composes `Panorama360View`, `DemoPanel`, `IntroModal`, back button
- Owns prev/next navigation logic: computes new index, updates Zustand, triggers camera
- Reads Zustand `activeHotspotId` and `currentHotspotIndex` to drive panel content

### `components/panorama/DemoPanel.tsx`
Slide panel showing hotspot detail (Client Component).
- Framer Motion spring animation: slides from bottom on mobile, from right on desktop
- Renders hotspot title, description, video (`<video>` element), external link
- Contains `NavigationArrows` for prev/next
- Close button dispatches `closePanel()` to Zustand

### `store/useSceneStore.ts`
Zustand store — the state bridge between Pannellum (imperative) and React (declarative).

```ts
activeHotspotId: string | null
currentHotspotIndex: number
panelOpen: boolean
// actions:
setActiveHotspotWithIndex(id, index)
openPanel() / closePanel()
clearActiveHotspot()
setCurrentHotspotIndex(n)
```

## Business Functions

### `lib/demos-db.ts`
All DB queries for business function demos.
- `getDemosForCategory(slug)`: single category
- `getDemosForAllCategories()`: parallel fetch for all 6 — used on landing
- `getDemoCountForCategory(slug)`: used for category card badges

### `lib/landing-categories.ts`
Static config for the 6 business function category cards (icon, label, route, color).
Hydrated with demo counts at render time.

### `components/business-function-cards.tsx`
3D flip card UI (CSS `perspective` + `rotateY`).
- Front: category image + icon + title
- Back: list of demo buttons
- Click demo → local state `selectedDemo` → opens `DemoDetailModal`

### `components/demo-detail-modal.tsx`
Modal showing full demo detail: narrative, tags, external link.
Triggered by client state — no route change.

## AI Workforce / Agents

### `mocks/agents.json`
Agent definitions (id, slug, name, role, department, capabilities, welcomeMessage…).
Currently the only source of agent data — `lib/agents.ts` reads this file.
**When DB integration is ready:** replace `lib/agents.ts` with Drizzle queries against `aiAgents` table.

### `lib/agents.ts`
Thin loader over `mocks/agents.json`.
- `getAgents()`: all agents
- `getAgentBySlug(slug)`: single agent

### `components/agent-chat.tsx`
Chat UI (Client Component).
- Renders agent avatar, status pill, welcome message, message history
- Input + submit appends user message then generates a mock assistant response
- **LLM integration point**: replace mock response generation here

## Styling

### `app/globals.css`
Tailwind v4 imports + all custom utilities.
- CSS custom properties: color tokens in oklch for light/dark modes
- `.perspective-1000`, `.backface-hidden`, `.rotate-y-180`: 3D flip card helpers
- `.panorama-hotspot`: hotspot marker base styles
- Keyframe animations: `pulse-ring`, `fade-in`, `slide-in-*`
- **Touch here carefully** — affects global layout and panorama hotspot appearance

### `app/layout.tsx`
Root layout. Loads fonts (Geist Sans/Mono, Playfair Display), wraps in `<Providers>`.

## Configuration

### `next.config.ts`
- Allows remote images from: `lh3.googleusercontent.com`, `images.unsplash.com`, `s3.amazonaws.com`
- Add new image domains here when integrating external media

### `tsconfig.json`
- Path alias `@/*` → project root
- Strict mode enabled
- Target: ES2017

### `components.json`
Radix UI / shadcn component registry config. Used when adding new UI primitives via CLI.

## Files to Treat with Extra Care

| File | Why |
|---|---|
| `auth.ts` | Controls who can access the entire application |
| `db/schema.ts` | Schema changes require coordinated migration + deploy |
| `components/panorama/Panorama360View.tsx` | Pannellum API + timing quirks — easy to break camera or hotspot behavior |
| `store/useSceneStore.ts` | Shared panorama state — changes affect viewer, panel, and navigation simultaneously |
| `app/globals.css` | Global — visual regressions can be broad and subtle |
